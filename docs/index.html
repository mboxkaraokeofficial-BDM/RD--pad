ภาพรวมโปรเจกต์

Project Overview — ทำไมถึงต้องสร้าง RD Pad

😤
Problem

ปัญหาของ MPAD2

ต้องดาวน์โหลดและติดตั้งแอพ บน iOS/Android ไม่รองรับทุกรุ่น ลูกค้าหลายคนไม่สะดวกติดตั้ง

🌐
Solution

RD Pad — Browser First

เปิดผ่านเบราว์เซอร์ได้เลย สแกน QR Code → เข้าใช้งานทันที รองรับทุก Platform ทุกอุปกรณ์

👥
Target Users

กลุ่มเป้าหมาย

ลูกค้าบ้าน (1 ห้อง) และร้านอาหาร/คาราโอเกะ (1–50 ห้อง) รองรับทั้ง Home User และ Business

📦
Compatible

รองรับอุปกรณ์

MBOX Echo Pro, MBOX Mini Spark ระบบปฏิบัติการ Android 11+ เชื่อมต่อผ่าน TCP Port 6666

ฟีเจอร์หลัก

5 หน้าหลัก — Navigation Tabs

🎵
Now Playing

เล่น

แสดงเพลงปัจจุบัน ปุ่ม Play/Pause/Skip ปรับ Key และระดับเสียง Floating mini player

🔍
Search

ค้นหา

ค้นหาเพลง Thai / EN / JP / KR จาก Cloud Song DB ดู Grid Cards เพิ่มเข้าคิวได้เลย

📋
Queue

คิว

จัดการคิวเพลง เรียงลำดับ ลบรายการ ดูรายชื่อเพลงที่จะเล่นต่อไป

✨
Effects

เอฟเฟกต์

Sound Effects ตบมือ หัวเราะ บูม Phase 4: DSP แต่งเสียงร้องได้

⚙️
Settings

ตั้งค่า

เชื่อมต่อเครื่อง MBOX เลือกห้อง Favorite Playlists Login ด้วย LINE/Google

Roadmap

แผนการพัฒนา 4 Phase

✅

Phase 1 — Full UI + Mock Data เสร็จแล้ว

สร้าง UI ครบทุกหน้า (เล่น / ค้นหา / คิว / เอฟเฟกต์ / ตั้งค่า) พร้อม Mock Data และ KaraokeController trigger layer

🔄

Phase 2 — Backend + TCP Bridge

Node.js WebSocket server เชื่อมต่อกับ MBOX จริงผ่าน TCP:6666 ส่ง command จริงได้

📋

Phase 3 — Auth + Cloud DB

Supabase Auth, LINE/Google OAuth, Cloud Song DB, Favorite Playlists per user, Multi-room shop support

🔮

Phase 4 — DSP Voice Processing

แต่งเสียงร้องแบบ Real-time DSP, Reverb, Echo, Pitch correction, Vocal enhancement

กลุ่มผู้ใช้

User Types — ออกแบบรองรับทั้งสองกลุ่ม

🏠

Home User

ลูกค้าบ้าน ห้องคาราโอเกะส่วนตัว 1 ห้อง

  • ไม่ต้อง Login ก็ใช้งานได้
  • สแกน QR → เชื่อมต่อทันที
  • Favorite Playlist (ต้อง Login)
  • ปรับ Key / Volume / MIC
  • Sound Effects
🏪

Shop / Restaurant

ร้านอาหาร ร้านคาราโอเกะ 1–50 ห้อง

  • Admin Panel จัดการทุกห้อง
  • ดู QR Code แต่ละห้อง
  • แสดงสถานะทุกเครื่อง
  • Staff Login (คนของร้าน)
  • รองรับ 1–50 ห้องต่อสาขา
